## Что такое CordisAI?

CordisAI — это платформа на основе искусственного интеллекта, которая автоматически анализирует эхокардиографию (УЗИ сердца). Работает просто: врач записывает видео ультразвукового исследования сердца пациента, загружает его в CordisAI, и через несколько секунд получает полный отчёт с измерениями, флагами заболеваний и визуальным анализом. Обычно такая работа занимает у кардиолога 15-30 минут.

**Простыми словами:** вы загружаете видео бьющегося сердца, и система говорит вам:
- Насколько хорошо сердце качает кровь (фракция выброса)
- Размеры камер сердца
- Толщину стенок сердца
- Есть ли признаки заболеваний сердца
- Готовый PDF-отчёт для врача

CordisAI **НЕ заменяет врача**. Это инструмент, который помогает врачам работать быстрее и точнее, предоставляя автоматические измерения как отправную точку.

## Зачем это нужно?

### Проблема
- **Эхокардиография** — самое распространённое кардиологическое исследование в мире. Только в США проводится более 30 миллионов в год
- **Ручной анализ** каждого исследования занимает 15-30 минут времени специалиста
- В мире **нехватка** обученных сонографов и кардиологов
- **Человеческий фактор** — два врача, измеряя одно и то же эхо, могут получить разные цифры (разброс между специалистами составляет 5-10%)
- В развивающихся странах во многих больницах есть аппараты УЗИ, но **нет специалистов** для интерпретации результатов

### Решение
CordisAI автоматизирует анализ:
- **Скорость**: результат за секунды вместо минут
- **Стабильность**: одно и то же видео всегда даёт одинаковые измерения
- **Доступность**: можно использовать в любой больнице с компьютером
- **Экономия**: снижает потребность в дорогостоящем времени специалистов для рутинных измерений

### Аналогичная платформа
CordisAI вдохновлён [us2.ai](https://us2.ai/) — сингапурской компанией, получившей одобрение FDA на автоматический анализ эхокардиографии. Наша платформа реализует аналогичный подход с использованием открытых данных и моделей.

## Как это работает?

Путь видео УЗИ сердца через CordisAI, шаг за шагом:

### Шаг 1: Загрузка
Врач загружает файл эхокардиографии. CordisAI принимает:
- **файлы .avi** — стандартный формат видео, используемый датасетом EchoNet-Dynamic
- **файлы .dcm** — формат DICOM, универсальный медицинский стандарт визуализации, используемый больницами по всему миру
- **директории CAMUS NIfTI** — исследовательский формат для кардиовизуализации

### Шаг 2: Классификация вида (View Classification)
ИИ сначала определяет, **какой вид сердца** показан на видео:
- **A4C (Apical 4-Chamber)** — апикальная четырёхкамерная позиция. Сердце видно снизу, видны все 4 камеры. Самый распространённый и полезный вид
- **A2C (Apical 2-Chamber)** — апикальная двухкамерная позиция. Сердце снизу, но под другим углом — видны левый желудочек и левое предсердие

Зачем это нужно: разные виды показывают разные части сердца, и измерения зависят от того, какой вид перед нами.

### Шаг 3: Сегментация ИИ
Это ядро CordisAI. Нейронная сеть **U-Net** смотрит на каждый кадр видео и определяет:
- **Left Ventricle (LV)** — левый желудочек, главная насосная камера (отмечается красным на наложении)
- **Myocardium** — миокард, сердечная мышца (отмечается жёлтым)
- **Left Atrium (LA)** — левое предсердие, верхняя левая камера, принимающая кровь из лёгких (отмечается синим)
- **Background** — фон, всё остальное

ИИ создаёт **маску** для каждого кадра — как бы закрашивает структуры, которые распознал. Это называется «семантическая сегментация».

### Шаг 4: Измерения
По маскам сегментации CordisAI рассчитывает 20+ кардиологических показателей:
- Сколько крови вмещает сердце (объёмы)
- Сколько крови выбрасывает за одно сокращение (фракция выброса)
- Толщина стенок
- Размеры камер
- И многое другое (полный список ниже)

### Шаг 5: Обнаружение заболеваний
На основе измерений CordisAI проверяет наличие признаков заболеваний сердца по медицинским рекомендациям (ASE 2015, ESC). Это **проверки на основе правил** — если показатель выходит за пределы нормы, соответствующее состояние помечается флагом.

### Шаг 6: Генерация отчёта
CordisAI создаёт:
- **PDF-отчёт** со всеми измерениями, флагами и графиками
- **FHIR R4 bundle** (стандартный медицинский формат данных для электронных медицинских карт)
- **DICOM Structured Report** (стандартный формат для радиологических заключений)

## Модели ИИ

CordisAI использует три нейросетевые модели. Вот что делает каждая и зачем.

### 1. Модель сегментации U-Net

**Что делает:** Смотрит на каждый кадр эхо-видео и определяет (сегментирует) структуры сердца — левый желудочек, миокард и левое предсердие.

**Архитектура:** U-Net 2D
- Изобретена в 2015 году специально для сегментации биомедицинских изображений
- Имеет форму буквы «U» — сначала сжимает изображение, чтобы понять общую картину (энкодер), затем разворачивает обратно в полный размер, чтобы пометить каждый пиксель (декодер)
- Имеет «пропускные соединения» (skip connections) между энкодером и декодером, чтобы мелкие детали не терялись
- Вход: чёрно-белый кадр 112x112 пикселей
- Выход: маска 112x112 пикселей с 4 классами (0=фон, 1=LV, 2=миокард, 3=LA)

**Результаты:** LV Dice score = 0.9153
- Dice score измеряет совпадение между предсказанием ИИ и правильным ответом (от врача)
- 1.0 = идеальное совпадение, 0.0 = нет совпадения
- 0.9153 означает 91.53% совпадения — это считается хорошим результатом для медицинской визуализации

**Почему U-Net?** Это золотой стандарт для медицинской сегментации изображений. Работает хорошо даже с ограниченным количеством обучающих данных и даёт точные границы.

**Статус обучения:** ГОТОВО

### 2. EF Regressor (EfficientNet-B0)

**Что делает:** Напрямую предсказывает фракцию выброса (EF) — сколько крови сердце выбрасывает за одно сокращение — по видео, без необходимости в сегментации.

**Архитектура:** EfficientNet-B0
- Современная эффективная сеть классификации изображений, разработанная Google
- Предобученная на ImageNet (миллионы обычных изображений) и дообученная на эхо-видео
- Берёт 32 равномерно распределённых кадра из видео и выдаёт одно число (EF в процентах)

**Результаты:** Лучшая MAE (средняя абсолютная ошибка) = 5.60%
- Это значит, что в среднем предсказание отличается от истинного EF на 5.6 процентных пунктов
- Для сравнения: разброс между врачами при оценке EF обычно составляет 5-10%

**Зачем две модели для EF?**
- Подход через сегментацию U-Net даёт больше измерений (объёмы, толщина стенок и т.д.) и является основным методом
- EF Regressor — это **запасной вариант** — если модель сегментации недоступна или не обучена, эта модель всё ещё может дать оценку EF
- В рабочем режиме, когда сегментация доступна, используется EF на основе сегментации

**Статус обучения:** ПРИОСТАНОВЛЕНО на эпохе 11/30 (MAE = 5.60%)

### 3. View Classifier (Классификатор видов)

**Что делает:** Определяет, какой вид показан на эхо-видео — A4C (4 камеры) или A2C (2 камеры).

**Архитектура:** EfficientNet-B0 (та же база, что и у EF Regressor)
- Бинарный классификатор: A4C (класс 0) vs A2C (класс 1)
- Берёт один кадр и выдаёт вероятности для каждого вида

**Результаты:** Точность на валидации = 93.3%

**Зачем это нужно:** Разные виды требуют разных подходов к измерениям. Пайплайн должен знать, какой вид перед ним, чтобы правильно рассчитать показатели.

**Статус обучения:** ГОТОВО (20 эпох)

## Датасеты (наборы данных)

### EchoNet-Dynamic

**Что это:** Публичный набор данных из 10 030 эхокардиографических видео из Стэнфордской университетской больницы.

**Что содержит:**
- Видео в апикальной четырёхкамерной позиции (A4C) в формате AVI
- У каждого видео есть эталонная фракция выброса (EF), измеренная кардиологами
- У части видео есть покадровая разметка границы LV в конце диастолы (ED) и систолы (ES)
- Видео чёрно-белые, 112x112 пикселей

**Для чего используем:**
- Обучение **EF Regressor** (прямое предсказание EF)
- Обучение **View Classifier** (как примеры A4C — 7 865 видео)

**Почему этот датасет?** Это самый большой публично доступный набор эхокардиографических данных с клиническими эталонными значениями. Стэнфорд выпустил его для продвижения ИИ-исследований в кардиологии.

**Ссылка:** Ouyang et al., "Video-based AI for beat-to-beat assessment of cardiac function," Nature, 2020.

### CAMUS (Cardiac Acquisitions for Multi-structure Ultrasound Segmentation)

**Что это:** Публичный набор данных 500 пациентов из Университетской больницы Сент-Этьена, Франция.

**Что содержит:**
- Оба вида (A4C и A2C) для каждого пациента
- Формат NIfTI (медицинский формат изображений)
- Эталонные маски сегментации, нарисованные кардиологами:
  - Левый желудочек (эндокард)
  - Миокард (сердечная мышца)
  - Левое предсердие
- Аннотированные кадры конца диастолы (ED) и конца систолы (ES)

**Для чего используем:**
- Обучение **модели сегментации U-Net** (это основной датасет для нашей ключевой модели)
- Обучение **View Classifier** (примеры A2C — 400 пациентов с A2C видами, плюс A4C примеры)

**Почему этот датасет?** Это единственный большой публичный набор данных, содержащий одновременно маски сегментации И несколько видов (A4C + A2C). Маски сегментации необходимы для обучения U-Net определять структуры сердца.

**Ссылка:** Leclerc et al., "Deep Learning for Segmentation using an Open Large-Scale Dataset in 2D Echocardiography," IEEE TMI, 2019.

## Какие показатели измеряет платформа?

Все показатели вычисляются автоматически по маскам сегментации ИИ. Ниже — что означает каждый из них, простым языком.

### Систолическая функция LV (Насколько хорошо сердце качает)

| Показатель | Полное название | Что это значит | Норма |
|---|---|---|---|
| **LVEF** | Left Ventricular Ejection Fraction | Процент крови, выбрасываемой за одно сокращение. Самый важный показатель работы сердца. Если сердце вмещает 100 мл и выбрасывает 60 мл — ваш EF = 60%. | 53-73% |
| **LVEDV** | LV End-Diastolic Volume | Сколько крови вмещает левый желудочек, когда полностью расслаблен (наполнен). Представьте это как размер «полного бака». | М: 62-150 мл, Ж: 46-106 мл |
| **LVESV** | LV End-Systolic Volume | Сколько крови остаётся после сжатия сердца. Это «остаток» после каждого сокращения. | М: 21-61 мл, Ж: 14-42 мл |
| **LVEDVi** | LVEDV, индексированный к размеру тела | LVEDV, делённый на BSA. Корректирует на размер тела — высокий человек естественно имеет более крупное сердце. | М: 34-74, Ж: 29-61 мл/м2 |
| **LVESVi** | LVESV, индексированный к размеру тела | LVESV, делённый на BSA. Та же идея — нормализация по размеру тела. | М: 11-31, Ж: 8-24 мл/м2 |
| **LVSV** | Stroke Volume (Ударный объём) | Количество крови, выбрасываемой за одно сокращение (LVEDV минус LVESV). | М: 35-95, Ж: 30-80 мл |
| **CO** | Cardiac Output (Сердечный выброс) | Общий объём крови, перекачиваемой за минуту (ударный объём x частота сердечных сокращений). Сколько крови ваше сердце доставляет организму каждую минуту. | 4.0-8.0 л/мин |

### Размеры LV и толщина стенок (Размер и структура)

| Показатель | Полное название | Что это значит | Норма |
|---|---|---|---|
| **IVSd** | Interventricular Septum at Diastole | Толщина перегородки между левым и правым желудочками в момент расслабления. Утолщение может указывать на гипертрофию. | 0.6-1.0 см |
| **LVIDd** | LV Internal Diameter at Diastole | Ширина левого желудочка в расслабленном состоянии. Увеличенный диаметр может указывать на дилатационную кардиомиопатию. | М: 4.2-5.9, Ж: 3.9-5.3 см |
| **LVIDs** | LV Internal Diameter at Systole | Ширина левого желудочка в момент сжатия. По сравнению с LVIDd показывает, насколько хорошо сердце сокращается. | М: 2.5-4.0, Ж: 2.2-3.5 см |
| **LVIDd index** | LVIDd, индексированный к размеру тела | LVIDd, делённый на BSA. Нормализует по размеру тела. | М: 2.2-3.1, Ж: 2.0-2.9 см/м2 |
| **LVIDs index** | LVIDs, индексированный к размеру тела | LVIDs, делённый на BSA. | М: 1.2-2.1, Ж: 1.1-1.9 см/м2 |
| **LVPWd** | LV Posterior Wall at Diastole | Толщина задней стенки левого желудочка. Утолщение = возможная гипертрофия. | 0.6-1.0 см |
| **RWT** | Relative Wall Thickness | Относительная толщина стенки. Формула: (IVSd + LVPWd) / LVIDd. Помогает классифицировать тип увеличения (концентрическое или эксцентрическое). | <= 0.42 |

### Масса LV

| Показатель | Полное название | Что это значит | Норма |
|---|---|---|---|
| **LVM** | LV Mass (Масса ЛЖ) | Вес мышцы левого желудочка, рассчитанный по толщине стенок и размеру камеры по кубической формуле ASE. Увеличенная масса = гипертрофия. | М: 88-224 г, Ж: 67-162 г |
| **LVMi** | LV Mass Index (Индекс массы ЛЖ) | Масса LV, делённая на BSA. Стандартный способ диагностики гипертрофии LV. | М: 49-115, Ж: 43-95 г/м2 |

### Глобальная деформация

| Показатель | Полное название | Что это значит | Норма |
|---|---|---|---|
| **GLS** | Global Longitudinal Strain | Измеряет, насколько сердечная мышца укорачивается при сокращении (в %). Отрицательное число — чем больше по модулю, тем лучше функция. GLS может обнаружить тонкие проблемы с сердцем до того, как упадёт EF. Примечание: наш GLS оценивается по контурам сегментации, а не по speckle-tracking (который требует сырых данных УЗИ). | от -20% до -16% |

### Левое предсердие

| Показатель | Полное название | Что это значит | Норма |
|---|---|---|---|
| **LA area** | Left Atrial Area (Площадь ЛП) | Размер левого предсердия на изображении. Увеличенное ЛП может указывать на повышенное давление в сердце (например, из-за клапанной болезни или сердечной недостаточности). | <= 20 см2 |
| **LAV** | Left Atrial Volume (Объём ЛП) | Объём левого предсердия, вычисленный из 2D-маски методом суммации дисков. Точнее, чем только площадь. | М: <= 58 мл, Ж: <= 52 мл |
| **LAVi** | LA Volume Index (Индекс объёма ЛП) | LAV, делённый на BSA. Стандартная мера увеличения ЛП. | <= 34 мл/м2 |

### Площадь поверхности тела

| Показатель | Полное название | Что это значит |
|---|---|---|
| **BSA** | Body Surface Area (Площадь поверхности тела) | Примерная общая площадь кожи (м2), вычисленная по росту и весу по формуле Мостеллера: BSA = sqrt(рост_см x вес_кг / 3600). Используется для «индексации» (нормализации) измерений по размеру тела, чтобы крупного и мелкого человека можно было корректно сравнить. |

### Классификации (не числа, а категории)

| Классификация | Что это значит |
|---|---|
| **EF Category** | Норма / Мягкое снижение (HFmrEF) / Умеренное снижение / Тяжёлое снижение (HFrEF) |
| **LV Geometry** | Нормальная геометрия / Концентрическое ремоделирование / Концентрическая гипертрофия / Эксцентрическая гипертрофия — описывает характер утолщения стенок |
| **GLS Category** | Норма / Мягко нарушен / Умеренно нарушен / Тяжело нарушен |

## Какие заболевания обнаруживает?

CordisAI использует **детекцию на основе правил** — проверяет, выходят ли показатели за пределы норм, определённых медицинскими рекомендациями (ASE 2015, ESC). Отдельная ИИ-модель для обнаружения заболеваний НЕ используется — медицинские правила применяются к показателям, рассчитанным ИИ.

### Сердечная недостаточность (Heart Failure)

**Что это?** Сердце не может перекачивать достаточно крови для нужд организма. Человек чувствует усталость, одышку, отёки ног.

**Как CordisAI определяет:**
- **HFrEF** (сердечная недостаточность со сниженной фракцией выброса): EF < 40% — сердце выбрасывает значительно меньше крови, чем в норме
- **HFmrEF** (сердечная недостаточность с мягко сниженной фракцией выброса): EF между 41-49% — пограничная работа сердца

**Клиническое значение:** Сердечной недостаточностью страдают более 60 миллионов человек в мире. Раннее обнаружение позволяет начать лечение, которое замедляет прогрессирование и улучшает качество жизни.

### Гипертрофия LV (LV Hypertrophy, LVH)

**Что это?** Сердечная мышца стала аномально толстой. Часто вызвано повышенным артериальным давлением (сердцу приходится работать сильнее, поэтому мышца растёт). Аналогия: как мышцы у спортсмена растут от нагрузки — так и сердце «накачивается» от хронического повышенного давления, но это нездоровый рост.

**Как CordisAI определяет:**
- Толщина стенки > 1.0 см (IVSd или LVPWd)
- Индекс массы LV > 115 г/м2 (мужчины) или > 95 г/м2 (женщины)
- **Концентрическая гипертрофия**: толстые стенки + большая масса (равномерное утолщение)
- **Эксцентрическая гипертрофия**: нормальное соотношение толщины стенок, но большая масса (расширение камеры)

**Клиническое значение:** Гипертрофия LV — сильный предиктор сердечно-сосудистых событий. При раннем обнаружении можно контролировать с помощью снижения артериального давления.

### Дилатация LV (LV Dilatation)

**Что это?** Левый желудочек аномально увеличен (растянут). Можно сравнить с воздушным шаром, который слишком сильно надули — стенки истончаются, и он хуже сжимается. Может быть вызвано хронической перегрузкой объёмом, клапанной болезнью или кардиомиопатией.

**Как CordisAI определяет:**
- LVIDd > 5.9 см (мужчины) или > 5.3 см (женщины)
- LVEDV значительно выше нормы

**Клиническое значение:** Расширенный LV менее эффективно качает кровь и может прогрессировать в сердечную недостаточность.

### Увеличение левого предсердия (LA Enlargement)

**Что это?** Левое предсердие аномально увеличено. Часто это признак хронического повышенного давления в сердце (например, из-за заболевания митрального клапана или диастолической дисфункции). Представьте: если «дверь» между предсердием и желудочком плохо работает, кровь застаивается, и предсердие растягивается.

**Как CordisAI определяет:**
- LAVi > 34 мл/м2

**Клиническое значение:** Увеличение ЛП связано с повышенным риском мерцательной аритмии (нерегулярное сердцебиение) и инсульта.

### Подозрение на амилоидоз (Amyloidosis Suspicion)

**Что это?** Кардиальный амилоидоз — это редкое, но серьёзное состояние, при котором аномальные белки откладываются в сердечной мышце, делая её жёсткой. Сердце перестаёт нормально расслабляться и наполняться кровью.

**Как CordisAI определяет (только скрининг):**
- Очень толстые стенки (IVSd > 1.2 см) в сочетании с
- Сниженным EF или нарушенным GLS
- Это флаг с НИЗКОЙ ДОСТОВЕРНОСТЬЮ — он предлагает дальнейшее обследование, а не является диагнозом

**Клиническое значение:** Амилоидоз часто пропускают, потому что он маскируется под другие состояния. Раннее обнаружение критически важно, так как новые препараты (тафамидис) могут замедлить прогрессирование.

## Архитектура и пайплайн

### Общая схема

```
Пользователь загружает эхо-видео (.avi или .dcm)
        |
        v
  [FastAPI Server] — порт 8002
        |
        v
  [Pipeline] — оркеструет все шаги
        |
        +---> [Video Loader] — читает и нормализует кадры видео
        +---> [View Classifier] — определяет вид A4C vs A2C
        +---> [U-Net Segmentation] — сегментирует LV, миокард, LA в каждом кадре
        +---> [Measurement Engine] — вычисляет LVEF, объёмы, толщину стенок и т.д.
        +---> [Disease Detection] — применяет медицинские правила к измерениям
        +---> [PDF Generator] — создаёт клинический отчёт
        +---> [FHIR Exporter] — создаёт стандартный медицинский пакет данных
        +---> [DICOM SR Generator] — создаёт радиологическое заключение
        |
        v
  Результаты возвращаются во фронтенд
        |
        v
  [React Frontend] — порт 3001
  Показывает: видео с ИИ-наложением, шкалу EF, таблицу измерений,
              флаги заболеваний, график сердечного цикла, кнопки скачивания
```

## Форматы экспорта

### PDF-отчёт
Готовый для врача PDF-документ, содержащий:
- Заголовок с информацией о пациенте
- Все измерения в организованных таблицах с нормальными диапазонами
- Флаги обнаружения заболеваний
- График площади LV в сердечном цикле
- Кривая деформации GLS (когда результат надёжен)
- Клинический отказ от ответственности (дисклеймер)

### FHIR R4 Bundle
[FHIR](https://hl7.org/fhir/) (Fast Healthcare Interoperability Resources) — международный стандарт электронного обмена медицинской информацией. CordisAI генерирует FHIR R4 Transaction Bundle, содержащий:
- Ресурс **Patient** (демография пациента)
- Ресурсы **Observation** (по одному на каждое измерение)
- Ресурс **DiagnosticReport** (сводка с заключениями)

Этот пакет можно напрямую отправить (POST) на любой FHIR R4 сервер:
- HAPI FHIR (open source)
- Azure Health Data Services
- AWS HealthLake
- Google Cloud Healthcare API

### DICOM Structured Report
[DICOM SR](https://www.dicomstandard.org/) — стандартный формат для радиологических и кардиологических заключений. CordisAI генерирует DICOM SR по шаблону TID 5200 (Echocardiography Procedure Report), содержащий все измерения и находки в формате, который напрямую интегрируется с больничными PACS (системами архивирования и передачи изображений).

## Словарь терминов

| Термин | Значение |
|---|---|
| **Echocardiography (Эхо)** | Ультразвуковое исследование (УЗИ) сердца |
| **Left Ventricle (LV)** | Левый желудочек — главная насосная камера сердца, которая выталкивает кровь в аорту и ко всем органам |
| **Left Atrium (LA)** | Левое предсердие — верхняя левая камера, принимающая насыщенную кислородом кровь из лёгких |
| **Myocardium** | Миокард — собственно сердечная мышца, которая сокращается и расслабляется |
| **Ejection Fraction (EF)** | Фракция выброса — процент крови, выбрасываемой из желудочка за одно сокращение. Главный показатель работы сердца |
| **End-Diastole (ED)** | Конец диастолы — момент, когда сердце полностью расслаблено и заполнено кровью (максимальный объём) |
| **End-Systole (ES)** | Конец систолы — момент, когда сердце максимально сжато (минимальный объём) |
| **Segmentation (Сегментация)** | Техника ИИ, при которой на изображении определяются и обводятся конкретные структуры (например, камеры сердца) |
| **U-Net** | Архитектура нейронной сети, разработанная специально для сегментации медицинских изображений |
| **BSA** | Body Surface Area — площадь поверхности тела, рассчитанная по росту и весу. Используется для нормализации показателей к размеру тела |
| **Simpson's Method** | Метод Симпсона — техника расчёта объёма камеры сердца из 2D-изображений путём разделения камеры на диски |
| **A4C** | Apical 4-Chamber — апикальная четырёхкамерная позиция, самый распространённый вид ЭхоКГ, показывает все 4 камеры сердца |
| **A2C** | Apical 2-Chamber — апикальная двухкамерная позиция, показывает левый желудочек и левое предсердие под другим углом |
| **Dice Score** | Мера совпадения между предсказанием ИИ и эталоном (0-1, чем выше тем лучше). Используется для оценки качества сегментации |
| **MAE** | Mean Absolute Error — средняя абсолютная ошибка, среднее отклонение предсказания от истинного значения |
| **DICOM** | Digital Imaging and Communications in Medicine — универсальный мировой стандарт для медицинских изображений |
| **FHIR** | Fast Healthcare Interoperability Resources — современный стандарт электронного обмена медицинскими данными |
| **PACS** | Picture Archiving and Communication System — больничная система хранения и передачи медицинских изображений |
| **ASE** | American Society of Echocardiography — Американское общество эхокардиографии, устанавливает нормы измерений |
| **ESC** | European Society of Cardiology — Европейское общество кардиологии, устанавливает клинические рекомендации |
| **HFrEF** | Heart Failure with Reduced Ejection Fraction — сердечная недостаточность со сниженной фракцией выброса (EF < 40%) |
| **HFmrEF** | Heart Failure with Mildly Reduced Ejection Fraction — сердечная недостаточность с мягко сниженной фракцией выброса (EF 41-49%) |
| **HFpEF** | Heart Failure with Preserved Ejection Fraction — сердечная недостаточность с сохранённой фракцией выброса (EF >= 50%) |
| **LVH** | Left Ventricular Hypertrophy — гипертрофия левого желудочка, аномальное утолщение сердечной мышцы |
| **GLS** | Global Longitudinal Strain — глобальная продольная деформация, измеряет укорочение сердечной мышцы |
| **RWT** | Relative Wall Thickness — относительная толщина стенки, коэффициент для классификации геометрии LV |
| **Cardiac Output** | Сердечный выброс — общий объём крови, перекачиваемый сердцем за минуту (л/мин) |
| **Stroke Volume** | Ударный объём — объём крови, выбрасываемый за одно сокращение (мл) |
| **Pixel Size** | Размер пикселя в реальных единицах (см). Необходим для перевода пиксельных измерений в сантиметры |
| **Inference (Инференс)** | Процесс использования обученной нейронной сети для получения предсказаний на новых данных |
| **Epoch (Эпоха)** | Один полный проход по всему обучающему набору данных при тренировке нейронной сети |

## Ограничения и дисклеймер

CordisAI — это **исследовательский прототип**. Важные ограничения:

1. **Нет одобрения FDA/CE** — нельзя использовать для клинической диагностики
2. **Только 2D** — не анализирует 3D-эхо или допплеровские данные
3. **Ограниченные виды** — только A4C и A2C (нет парастернальных, субкостальных или супрастернальных)
4. **Нет допплер-анализа** — не может измерить скорости кровотока (E/A ratio, E/e' и т.д.)
5. **Нет анализа правого желудочка** — правый желудочек не сегментируется (запланировано в Фазе 2)
6. **GLS приблизительный** — получен из анализа контуров, а не из настоящего speckle-tracking
7. **Предвзятость обучающих данных** — модели обучены на конкретных популяциях и оборудовании
8. **Все результаты требуют проверки врачом** — измерения ИИ — это отправная точка, а не финальный ответ
